Что такое "Protractor"?

Protractor - фреймворк комплексного тестирования, с помощью которого можно тестировать Angular приложение прямо в браузере, имитируя взаимодействия, как c реальным пользователем.

По умолчанию с ним в коробке идет 2 фреймворка для тестирования:

  • Юнит тесты на Jasmine и Karma
  • Комплексные тесты на Protractor

Основное различие - первый используется для тестирования логики компонентов и сервисов, а второй используется для проверки функциональности высокого уровня приложения (в том числе и UI элементы).

В каком файле нужно подключать сторонние скрипты и стили и почему?

Cторонние скрипты и стили должны подключаться в angular.json, а не в index.html как это делается в верстке, например.

Почему именно так? Важно, чтобы все настройки были в одном месте, а не размазаны по всему приложению. Причем файл angular.json как раз для этого и предназначен, т.к. является главным конфигурационным файлом Angular Workspace.

Зачем нужно свойство "bootstrap" в декораторе "@NgModule()"?

В свойстве bootstrap указывается компонент, который будет использоваться в качестве основного при загрузке модуля.

Что такое "Angular CLI"?

Angular CLI - это npm-модуль, реализующий интерфейс командной строки для создания, разработки и поддержки Angular приложений. В системе он должен быть установлен глобально.

Что такое модуль в Angular?

Angular модуль - это класс c декоратором @NgModule(), который служит изолирующей логической объединяющей структурой для компонентов, директив, фильтров и сервисов. 

@NgModule({{'{'}}
	declarations: [ParentComponent], // дочерние компоненты, директивы и фильтры
	imports: [SomeModule], // другие модули, используемые в текущем модуле
	providers: [SomeService], // импортирование в модуль необходимых сервисов приложения
	bootstrap: [ParentComponent], // имя главного компонента модуля
	exports: [SomeComponent], // экспортируемые компоненты, директивы и фильтры, которые могут быть использованы другими модулями
	{{'}'}})
export class ParentModule {{'{}'}} // класс, к которому привязывается декоратор @NgModule()
Что такое компонент в Angular?

Angular компонент - это часть интерфейса приложения с собственными шаблоном, логикой и стилями. Причем последние два являются необязательными.

Все компоненты в совокупности и есть Angular App.

За создание компонента отвечает декоратор @Component(). 

@Component{{'({'}}
	selector: 'app-root', // название компонента
	templateUrl: './app.component.html', // путь к HTML-файлу. Является обязательным свойством
	styleUrls: ['./app.component.scss'], // массив путей к SCSS-файлам
{{'})'}}
export class AppComponent {{'{}'}} // класс, к которому привязывается декоратор @Component()
Зачем нужны сервисы в Angular?

Сервисы нужны для предоставления данных компонентам. Причем в сервисах мы можем хранить не только запросы к серверу, но и различные преобразования данных.

Зачем нужны директивы в Angular?

Предназначение директив - преобразование DOM-элементов нужным нам образом.

По своей реализации директивы практически идентичны компонентам, т.к. компонент - это директива с HTML-шаблоном.

Директивы создаются с помощью декоратора @Directive() с конфигурационным объектом и подразделяются на 3 типа:

  • С собственным шаблоном - это компоненты (компоненты являются директивами, см.выше);
  • Структурные - изменяют структуру DOM-дерева (*ngIf, *ngFor и все другие, которые начинаются со "*");
  • Атрибуты - изменяют внешний вид или поведение по умолчанию элемента DOM-дерева.
Какие существуют механизмы компиляции приложения в Angular?

Начиная с 9-ой версии Angular, механизм Angular compile реализован в двух режимах:

  • JIT-компиляция (Just-in-TIme) - приложение компилируется в момент его запуска в браузере. Используется по умолчанию;
  • AOT-компиляция (Ahead-of-Time) - компиляция происходит в момент сборки приложения.

В режиме разработки рекомендуется использовать JIT-компиляцию, т.к. сборка приложения занимает меньше времени. Если же необходима итоговая версия приложения, то рекомендуется испольльзовать AOT-компиляцию.

При сборке приложения с флагом --prod AOT-компиляция используется по умолчанию.

Какие существуют механизмы привязки данных?

В Angular существует 4 способа привязки данных:

  • интерполяция - данные берутся из компонента;
  • одностороннее связывание - данные берутся из компонента;
  • обработка событий - данные после определенного события возвращаются в компонент;
  • двустороннее связывание - данные берутся из компонента и после определенного события ему же возвращаются.
Интерполяция

Примеры интерполяции:

<p>Hello, {{name}}</p> // (1) Основной способ использования интерполяции
<input type="text" name={{name}}> // (2) Так тоже будет работать, но такая запись редко встречается

При использовании интерполяции данные для переменной name берутся из того же самого компонента, к которому привязан шаблон с текущей разметкой. Данный способ привязки данных обычно используется, чтобы вывести в содержимом разметки какие-то данные.

Одностороннее связывание

Если необходимо изменять данные атрибутов элементов разметки, то интерполяцию тоже можно использовать (см. выше пример (2)), но в основном это делают через одностороннее связывание. Но суть та же: 

<input type="text" [name]="name"> // (2*) запись второго примера из интерполяции через одностороннее связывание
<input type="text" bind-name="name"> // (2**) Альтернативная запись. Встречается реже
Обработка событий

Что касается событий, то в Angular все события можно отследить и обработать , привязав к ним вызов метода класса. Делается это следующим образом: 

<button (click)="showItems()">Я кнопка</button>
<button on-click="showItems()">Я кнопка-близнец</button> // альтернативный вариант записи

В данных примерах при клике на кнопку будет вызвана функция showItems(). Названия событий всегда должны быть заключено в круглые скобки или предваряться префиксом on-.

Если необходимо получить подробную информацию о событии, то она хранится в объекте $event, который передается в функцию в качестве аргумента. 

<button (click)="showItems($event)">Я кнопка</button>

Помимо встроенных событий можно создавать также и свои - пользовательские события. За этот функционал отвечает класс EventEmitter.

Двустороннее связывание

Механизм двустороннего связывания (two way binding) используется, когда необходимо отобразить свойство в шаблоне (одностороннее связывание) и обновить его при изменении значения (обработка события) без перезагрузки страницы.

Синтаксис двустороннего связывания представляет собой слияние одностороннего связывания ("[]") и привязки события ("()"). 

<some-item [(name)]="myName"></some-item>

Запись [(name)]="myName" означает, что при изменении name в дочернем компоненте (откуда оно будет получено в измененном виде), измененнное значение будет присвоено свойству myName компонента some-item.

Как реализовывается двухстороннее связывание в полях формы?

Отличительной особенностью полей форм в механизме двухстороннего связывания является использование директивы NgModel, которая может использоваться только с элементами формы. 

<input type="text" [(ngModel)]="value" />

При изменении значение в инпуте, оно записывается в свойство value. Аналогично и наоборот, если значение придет из компонента, то оно так же подставится в инпут.

Какая разница между декораторами "@Input()" и "@Output()"?

Компоненты Angular могут принимать от родителя данные и так же их отдавать. За это как раз и отвечают декораторы @Input() и @Output(), где исходя из названия становится понятно, что:

  • @Input() - принимает входящие данные (пропсы);
  • @Output() - отдает родителю данные при срабатывании какого-то события.
Расскажите про жизненный цикл компонента (Angular hooks)?

Каждый компонент имеет свой жизненный цикл, где на каждом этапе цикла вызываются соответствующие методы, так называемые хуки (Angular Hooks).

Ниже перечислены все методы жизненного цикла компонента в порядке их вызова:

  • OnChanges - устанавливаются или изменяются значения входных свойств класса компонента;
  • OnInit - устанавливаются свойства самого компонента. Данный метод вызывается единожды вслед за первым вызовом OnChanges();
  • DoCheck - происходит изменение свойства или какое-то событие;
  • AfterContentInit - в шаблон включается контент, заключенный между тегами компонента (в дочернем компоненте это содержимое подставится вместо <ng-content></ng-content> );
  • AfterContentChecked - аналогичен DoCheck(), только используется для контента, заключенного между тегами компонента (происходит проверка на изменение содержимого, которое подставляется вместо <ng-content></ng-content> );
  • AfterViewInit - инициализируются дочерние компоненты, указанные в шаблоне текущего компонента;
  • AfterViewChecked - аналогичен DoCheck(), только используется для отслеживания изменений в дочерних компонентах;
  • OnDestroy - компонент удаляется из DOM-дерева.

Angular hooks реализованы в виде интерфейсов, реализующих функции, совпадающие по названию с названием интерфейса + префикс ng: 

export class ContactsItemComponent implements OnInit {{'{'}}
	ngOnInit() {{'{'}}
		console.log('OnInit')  // OnInit
	{{'}'}}
{{'}'}}

Также при написании кода считается хорошим тоном указывать у класса реализацию того или иного метода, используемого в нем. В примере выше это implements OnInit
