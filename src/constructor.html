Каждый компонент имеет свой жизненный цикл, где на каждом этапе цикла вызываются соответствующие методы, так называемые хуки (Angular Hooks).

Ниже перечислены все методы жизненного цикла компонента в порядке их вызова:

  • ngOnChanges - вызывается в момент установки или изменения одного или нескольких значений входных свойств класса компонента. Метод аргументом принимает объект SimpleChanges, в котором хранятся текущее и предыдущее значения свойств. Если у компонента есть свойства, входящиее извне через декоратор @Input, срабатывает перед хуком ngOnInit. Если нет входящих свойств, то данный хук вызываться не будет;
  • ngOnInit - на данном этапе устанавливаются свойства самого компонента. Вызывается единожды, после первого вызова ngOnChanges, если есть входные свойства. Если нет входных свойств, вызывается первым;
  • ngDoCheck - происходит изменение одного или нескольких свойств или какое-то событие. Вызывается сразу после ngOnChanges при каждом запуске обнаружения изменений, и сразу после ngOnInit при первом запуске;
  • ngAfterContentInit - в шаблон включается контент, заключенный между тегами компонента (в дочернем компоненте это содержимое подставится вместо <ng-content></ng-content> );
  • ngAfterContentChecked - аналогичен DoCheck(), только используется для контента, заключенного между тегами компонента (происходит проверка на изменение содержимого, которое подставляется вместо <ng-content></ng-content> );
  • ngAfterViewInit - инициализируются дочерние компоненты, указанные в шаблоне текущего компонента;
  • ngAfterViewChecked - аналогичен DoCheck(), только используется для отслеживания изменений в дочерних компонентах;
  • ngOnDestroy - компонент удаляется из DOM-дерева.

Angular hooks реализованы в виде интерфейсов, реализующих функции, совпадающие по названию с названием интерфейса + префикс ng: 

export class ContactsItemComponent implements OnInit {
		ngOnInit() {
			console.log('OnInit')  // OnInit
		}
	}

Также при написании кода считается хорошим тоном указывать у класса реализацию того или иного метода, используемого в нем. В примере выше это implements OnInit.
